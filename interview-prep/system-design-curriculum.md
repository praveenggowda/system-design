# System Design Curriculum

A structured map of every concept area to master for any system design question.

| # | Area | Topics | Tools | Covered |
|---|---|---|---|---|
| 1 | System Design Fundamentals | What is System Design, Monolith vs Microservices, Horizontal vs Vertical Scaling, Latency vs Throughput, CAP Theorem and Trade-offs | | CAP Theorem (Real-Time Balance — CP), Horizontal scaling (all designs), Latency vs Throughput (all designs) |
| 2 | Load Balancing and Traffic Management | Load Balancing Algorithms, Global Load Balancers, DNS-based Load Distribution, Rate Limiting, Circuit Breakers | NGINX, AWS ELB / GCP Load Balancer, Envoy Proxy, Cloudflare | Rate Limiting — Fixed Window, Sliding Window, Token Bucket (Rate Limiter design + live coding), Circuit Breaker (live coding project) |
| 3 | Caching and CDN Strategies | Write-through and Write-around Cache, Cache Invalidation, CDN for Static Content, Cache Hierarchy, TTL and Eviction Policies | Redis, Varnish, Fastly, Akamai, Cloudflare CDN | Write-through cache (Real-Time Board, Banking Ledger), Cache Invalidation via Redis Pub/Sub (Real-Time Balance), Per-item cache invalidation (Real-Time Board), LRU Cache (URL Shortener + live coding), TTL (all Redis designs) |
| 4 | API Design Principles | REST vs gRPC vs GraphQL, WebSockets and Long Polling, Message Queues (Async), Protocol Buffers | Postman, Swagger, gRPC, Apache Kafka, RabbitMQ, Redis | REST (all designs), WebSockets (Real-Time Board, Chat, Item Reordering), Message Queues — Kafka, SQS, SNS (all designs), 202 Accepted async pattern (Payment System) |
| 5 | Database and Storage Design | SQL vs NoSQL, Sharding and Replication, Caching Strategies, Consistency Models, Backup and Recovery | PostgreSQL, MySQL, MongoDB, Cassandra, Redis, Memcached, Amazon RDS, DynamoDB | SQL vs NoSQL trade-offs (all designs), Sharding by boardId/accountId (Banking Ledger, Real-Time Board), Read Replicas (Real-Time Board, Banking Ledger), Tiered storage — hot PostgreSQL + cold S3 Parquet + Athena (Real-Time Board, Metrics), DynamoDB (Automation, Webhook Delivery), Double-entry bookkeeping (Banking Ledger — extra) |
| 6 | Communication Between Services | Event-Driven Architecture, CQRS and Event Sourcing, Saga Pattern, Backend for Frontend (BFF), Service Mesh | Apache Kafka, Apache Flink, Istio, NATS, Temporal.io | Event-Driven Architecture (Automation, NaaS, Event Processor), CQRS (Banking Ledger, Automation), Outbox Pattern (Banking Ledger, Notification, Real-Time Board, Event Processor), Saga Pattern — Choreography and Orchestration (Payment System), SNS fan-out to SQS (Notification, Event Processor), Dead Letter Queue + Kafka DLT (all designs — extra), At-least-once delivery + idempotent consumers (all designs — extra) |
| 7 | Scalability and Fault Tolerance | Auto-scaling, Failover Systems, Health Checks and Heartbeats, Leader Election, Redundancy Planning | Kubernetes, HPA, AWS Auto Scaling, Consul, Zookeeper, Terraform | Multi-instance services with failover (all designs), Redundancy via read replicas and multiple service instances (all designs). Leader Election, Kubernetes, Zookeeper not yet covered. |
| 8 | Security and Reliability | Secure API Design, Authentication and Authorization, Encryption in Transit and At Rest, Rate Limiting and Abuse Prevention, Chaos Engineering | OAuth2 / JWT, HashiCorp Vault, Open Policy Agent, Chaos Monkey, Snyk | Idempotency keys (Payment System, Booking System, Webhook Delivery), Rate Limiting (Rate Limiter design), Auth via identity service (all designs — boundary only). OAuth2/JWT deep dive, HashiCorp Vault, Chaos Engineering not yet covered. |
| 9 | Observability and Monitoring | Logging, Metrics, Tracing, Distributed Tracing (OpenTelemetry), Alerting Systems, Real-time Dashboards, Log Aggregation | Grafana, ELK Stack, Datadog, New Relic | Full Metrics and Logging system design (Kafka + Elasticsearch + InfluxDB + Grafana + Kibana + S3 cold storage), Structured logging with trace_id (all designs), Observability sections in every design (metrics, alerts, dashboards), Datadog + Grafana + CloudWatch + Kibana (NaaS production background — extra) |
| 10 | Real-World System Design Patterns | Outbox Pattern, Saga Pattern, CQRS, Event Sourcing, Idempotency, Rate Limiting, Circuit Breaker, Bulkhead, Sidecar / Service Mesh, Fan-out on Write vs Fan-out on Read, Two-Phase Commit, Distributed Locking | | Outbox Pattern ✅, Saga Pattern ✅, CQRS ✅, Idempotency ✅, Rate Limiting ✅, Circuit Breaker ✅, Fan-out on Write vs Fan-out on Read ✅ (News Feed, Notification), Distributed Locking ✅ (Booking System), Optimistic Locking ✅ (Item Reordering, Real-Time Board — extra), Payment State Machine ✅ (Payment System — extra). Two-Phase Commit, Bulkhead, Service Mesh not yet covered. |
